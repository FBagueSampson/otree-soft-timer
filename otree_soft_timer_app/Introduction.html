This app demonstrates the use of a JavaScript-based soft-timer for oTree.

The first section of this app demonstrates a {{ C.time_on_multispan_task }}-second timer that spans multiple pages. The second section demonstrates a {{ C.time_on_per_page_task }}-second, per-question timer. Code for clearing the browser's sessionStorage is provided on the final page.

Feature overview:
  • Each page/submit can have its own timer or the timer can span multiple pages
  • The soft-timer does NOT reset on page refresh
  • Each subsection/page class of an experiment can have its own timer
  • Timer can be set for hours, minutes, or just seconds; its display adjusts accordingly
  • Two different (and entirely optional) user-notification methods
  • Primary features are controlled in the __init__.py; look-and-feel settings are adjusted via stylesheet (included at the page here)

How it works:
The soft timer works by storing the time remaining in the browser's "sessionStorage." To see what is being recorded by the browser, press Ctrl + Shift + i, then navigate to the "Storage" tab/section of the inspector, and find "Session Storage." Please note that this approach to a soft-timer has NOT been tested for use with MTurk, where experimenter control over the local machine is non-existent.

User notification of time expiration:
Optionally, users can be shown an on-page message that appears upon the timer's expiration, then persists until the timer is next reset. Additionally (or alternatively), subjects can receive a one-time pop-up alert notifying them the instant that the timeout happens. Once acknowledged (the user clicks "Ok"), the pop-up message will not reappear until the timer is reset. The behavior of each timer's timeout messages is set in the class MyPage(Page) of your app's __init__.py file.
